## 项目主要需求

### 项目概述

【Web前端】家居装饰风格测试与灵感收集平台。纯前端装修风格探索工具，用户通过风格问卷测试确定偏好，浏览不同风格的案例图集，收集灵感图片到灵感板，对比不同方案。数据存储于 localStorage / IndexedDB。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个数据模型）

1. **风格画像（StyleProfile）** — 字段：用户ID、测试结果（各风格得分 Map<string, number>）、主风格（Modern/Chinese/Nordic/Japanese/Industrial/Mediterranean/Minimalist）、辅风格、测试时间、是否已完成测试
   - 定义位置：`types/profile.ts`

2. **灵感图片（InspirationImage）** — 字段：图片ID、图片URL、所属风格、房间类型（LivingRoom/Bedroom/Kitchen/Bathroom/Balcony/Study/Entrance）、标签列表、来源描述、收藏时间
   - 定义位置：`types/inspiration.ts`

3. **灵感板（MoodBoard）** — 字段：灵感板ID、名称、描述、创建时间、封面图URL、包含的图片ID列表、标签列表
   - 定义位置：`types/moodboard.ts`

4. **风格问卷题目（QuizQuestion）** — 字段：题目ID、题目文本、图片URL（可选）、选项列表（选项文本+对应风格权重Map）、排序
   - 定义位置：`types/quiz.ts`

5. **对比方案（ComparisonPlan）** — 字段：方案ID、方案名称、灵感板ID、风格标签列表、颜色方案列表（hex数组）、创建时间
   - 定义位置：`types/comparison.ts`

### 核心页面（≥ 4 个）

1. **风格测试** `/quiz` — 消费：QuizQuestion + StyleProfile
   - 图文并茂的问卷流程（每题配风格参考图），进度条，提交后生成风格雷达图和推荐风格描述
   - 使用 `<QuizCard>`, `<ProgressBar>`, `<StyleRadarChart>`

2. **灵感图集** `/gallery` — 消费：InspirationImage + StyleProfile
   - 瀑布流图片展示（按风格和房间类型筛选），双击/长按收藏到灵感板
   - 使用 `<ImageCard>`, `<FilterTabs>`, `<EmptyState>`

3. **灵感板** `/moodboards` — 消费：MoodBoard + InspirationImage
   - 灵感板列表（卡片+缩略图），灵感板详情（拖拽排列图片），新建灵感板
   - 使用 `<MoodBoardCard>`, `<DraggableGrid>`, `<EmptyState>`

4. **方案对比** `/compare` — 消费：ComparisonPlan + MoodBoard
   - 左右对比两个方案的灵感板、颜色方案、风格标签，支持多方案并列
   - 使用 `<ComparisonPanel>`, `<ColorSwatch>`, `<StyleTag>`

5. **个人风格档案** `/profile` — 消费：StyleProfile + MoodBoard
   - 风格雷达图（ECharts radar）、历史测试结果、收藏的灵感板列表、风格匹配推荐
   - 使用 `<StyleRadarChart>`, `<MoodBoardCard>`

### 横切关注点

1. **全局异常处理**
   - 触达文件层：图片加载失败fallback + IndexedDB异常 + ErrorBoundary + 错误类型定义

2. **主题切换（深色/浅色）**
   - 触达文件层：CSS变量 + themeStore + 组件适配 + 主题类型定义

### 共享枚举/常量（≥ 2 组）

1. **装修风格 DecorStyle**：Modern / Chinese / Nordic / Japanese / Industrial / Mediterranean / Minimalist
   - 前端：`types/enums.ts`

2. **房间类型 RoomType**：LivingRoom / Bedroom / Kitchen / Bathroom / Balcony / Study / Entrance

3. **风格颜色常量 STYLE_COLORS** — 每种风格对应的品牌色
   - 前端：`constants/styleColors.ts`

### 共享前端组件清单（≥ 3 个）

1. **`<ImageCard>`** — 灵感图集、灵感板详情
2. **`<FilterTabs>`** — 灵感图集（风格/房间筛选）
3. **`<StyleRadarChart>`** — 风格测试结果、个人档案
4. **`<ProgressBar>`** — 风格测试进度
5. **`<EmptyState>`** — 所有空状态

### 共享 hooks/utils 清单（≥ 2 个）

1. **`useQuiz`** — 封装问卷流程（答题→计算权重→生成风格画像）
2. **`useLocalStorage`** — 通用本地存储hook
3. **`styleCalculator`** — 风格得分计算工具（加权求和、归一化、排序）

### 技术要求

- **前端端口**：18806
- **前端**：Vue 3 + TypeScript + Vite
- **UI 库**：Tailwind CSS
- **图表**：ECharts（雷达图）
- **拖拽**：vue-draggable-plus
- **瀑布流**：自定义 CSS columns 或 vue-masonry
- **状态管理**：Pinia
- **本地存储**：IndexedDB（Dexie.js）
- **路由**：Vue Router 4
- **构建工具**：Vite
